The Problem

200+ projects, 400+ engineers, flattened revenue — why?

  • Allocation mismatches — staffing based on negotiation and relationships, not skills
  • Over-allocation — burnout, quality issues, hidden unbillable time
  • No capacity visibility — can't answer "can we staff this?" with confidence
  • Revenue leakage — roles unstaffed = billable hours lost
  • 3-5 days to staff — email chains, manual spreadsheet matching
  • Hidden talent missed — only considers people you already know
  • No forward view — no demand forecasting, reactive only

Core Challenge: Transition from subjective, negotiation-based staffing to a predictive, evidence-driven model.

Our Solution

AI-driven framework: right people, right projects, right time

Before (Manual)

  • Email-based negotiation
  • 3-5 days to staff
  • Know-who bias
  • No forecasting
  • Reactive only
  • Grade-based, not skill-based

After (RMG Engine)

  • AI-parsed PDF forms → auto-route
  • 3 seconds to recommend
  • Skill + semantic + competency matching
  • 6-month demand forecasting
  • Proactive gap detection (191 found)
  • Explainable — rationale per candidate

Outcome: Maximise billable utilisation while protecting delivery quality — decision-support outputs, not just dashboards.

AI Recommendation Pipeline

8-step process — from request to ranked candidates in seconds

1. COE Detection
SQL frequency → GPT fallback
2. Skills Extraction
GPT infers when null
3. Semantic Match
Embeddings cosine similarity
4. Formula Scoring
Weighted sum, 643 employees
5. Rationale
GPT-4o 2-3 sentence per candidate
6. LLM Re-Ranking
Holistic reorder top 10
7. KB Proof
Past project evidence
8. Hire Signal
When no match exists

Key: Steps 1-4 use database + embeddings (fast, free). Steps 5-8 use GPT-4o (intelligent, ~$0.015/role).

Scoring Formula

Transparent, auditable, explainable scoring

With competency:

total = skill × 0.40 + comp × 0.25 + avail × 0.25 + prod × 0.10

Without competency:

total = skill × 0.65 + avail × 0.25 + prod × 0.10

Skill blend: 50% COE assessment + 50% semantic similarity

Available

Free capacity ≥ needed %

Best Match

Allocated, score ≥ 40%

Stretch

Weak fit — filtered out

Demo: Engine — Extensions

Automatically detects resource gaps before they become problems

What it detects

Resources whose allocation ends before their project ends — creating a staffing gap that needs filling.

Logic: allocation.end_date < project.project_end_date

191 gaps detected

What it does

  • Groups by project — who's leaving and when
  • Shows days until gap opens
  • Click any leaving resource → AI recommends replacement
  • Same scoring engine as Pipeline

Proactive vs Reactive: Previously, gaps were only noticed when someone emailed. Now the system surfaces them automatically.
