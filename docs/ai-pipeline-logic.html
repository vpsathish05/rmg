Pre-computed Employee Embeddings

Each employee's full skill profile (all their skills, COEs, scores) is embedded into a 1536-dimension vector during ETL (build_skill_embeddings.py). Stored in employee_skill_embeddings table. This only needs to be rebuilt when skill data changes.

4 Formula Scoring

Scores ALL eligible employees using 4 weighted components. Pure math — no AI calls.

Final Formula

With competency:
  total = skill x 0.40 + competency x 0.25 + availability x 0.25 + productivity x 0.10

Without competency:
  total = skill x 0.65 + availability x 0.25 + productivity x 0.10

Skill blend:
  skill_score = 50% x COE_assessment + 50% x semantic_similarity

Category Assignment

Available

Employee has free capacity >= requested allocation %. Ready to deploy.

Best Match

Fully allocated BUT total_score >= 0.40. Strong fit — discuss reallocation.

Stretch

Allocated AND weak fit (score < 0.40). Not shown in recommendations.

How EMP11 was scored

  1. COE Detection No "Senior Solution Consultant" in skills DB → SQL global fallback → Data Engineering
  2. Skills Present in pipeline data → "PII handling, CDC, Scalable execution" (no LLM needed)
  3. Semantic Match Embedded role query → compared 286 profiles → EMP11 similarity = 51%
  4. Formula Scoring COE skills: avg([4,4,4,4,4]) / 5 = 0.80
    Blended skill: 50% x 0.80 + 50% x 0.51 = 0.655
    Availability: 0% allocated = 1.00
    Productivity: 166h / 320h = 0.52
    Total: 0.655 x 0.65 + 1.00 x 0.25 + 0.52 x 0.10 = 73%
  5. Category 100% free >= 100% needed → Available
  6. Re-rank GPT-4o moved EMP529 (Solutions Consultant) up — closer role title match
